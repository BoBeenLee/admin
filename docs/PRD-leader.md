## 1. 개요

| 항목 | 내용 |
|------|------|
| **페이지명** | 리더 팀 출석부 |
| **파일** | `leader.html` |
| **역할** | 리더(튜터/진행자)의 팀 배정, 출석 상태, 대타, 불참 일정을 종합 관리하는 출석부 |

## 3. 기능 요구사항

### 3.1 팀 탭 네비게이션

| ID | 기능 | 설명 | 우선순위 |
|----|------|------|----------|

| TAB-02 | 영어 팀 탭 | 영어 전체 / 월11시 / 월8시 / 화11시 / 화8시 / 수11시 / 수8시 / 목11시 / 목8시 / 토11시 / 일11시 | P0 |

| TAB-04 | 활성 탭 스타일 | 선택 탭: 검정 배경 + 흰 글씨, 비선택: 흰 배경 + 보더 | P0 |

## 4. 데이터 모델

### 4.1 리더 데이터 (Mock, 60명 생성)

```
Leader {
  name: string              // 이름 (고유)
  phone: string             // 010-XXXX-XXXX
  age: number               // 20-42세
  lv: number (1-5)          // 리딩 레벨
  teams: string[]           // 배정 팀 (1-2개)
  readingLevels: {team: lv} // 팀별 리딩 레벨 (독립)
  att: number               // 출석 횟수
  abs: number               // 불참 횟수
  sub: number               // 대타 횟수
  total: 8                  // 총 회차
  rate: number              // 출석률 %
  blocks: Block[8]          // 8회차 출석 블록
  todayChecked: boolean     // 금일 출석 여부
  memo: string              // 메모
  schedule: {status: Entry[]} // 상태별 일정 데이터
}
```

### 4.2 출석 블록

```
Block {
  s: string          // 상태: 출석완료 | 불참 | 대타출석완료 | 출석예정
  bg: string         // 배경색 클래스
  border: string     // 보더 클래스
  color: string      // 텍스트색 클래스
  date: {d, w}       // 날짜: {d: "04.01", w: "월"}
  members: string    // 참여 멤버 (또는 "—")
}
```

### 4.3 출석률 색상 규칙

| 출석률 | 색상 | 클래스 |
|--------|------|--------|
| >= 70% | 초록 | `text-emerald-600` |
| 40-69% | 주황 | `text-amber-600` |
| < 40% | 빨강 | `text-red-600` |

### 4.4 팀 색상 규칙

| 언어 | 색상 | 배경 |
|------|------|------|
| 영어 | `#9B59B6` (보라) | `#f5eafa` |
| 일어 | `#007BFF` (파랑) | `#e6f0ff` |

## 5. 인터랙션 명세

| 인터랙션 | 트리거 | 동작 |
|----------|--------|------|
| 팀 탭 전환 | 탭 클릭 | 해당 팀 리더만 필터링, 활성 스타일 변경 |
| 날짜 탭 전환 | 탭 클릭 / 스와이프 / 화살표 | 해당 날짜의 리더 상태 렌더링 |
| 언어/시간 필터 | 필터 버튼 클릭 | 즉시 필터 적용 (활성 스타일 토글) |
| 상태 필터 | 요약바 버튼 클릭 | 멀티셀렉트 토글, 즉시 필터 적용 |
| 불참 해결 | 체크박스 클릭 | 취소선 + 초록 전환, 전체 해결 시 행 제거 (0.4s 페이드) |
| 검색 | 입력 | 실시간 필터링 (이름/전화번호) |
| 정렬 변경 | 셀렉트 변경 | 테이블 재정렬 |
| 금일 출석 토글 | 체크 버튼 클릭 | 회색 ↔ 초록 토글 |
| 메모 편집 | 클릭 → 타이핑 → 포커스아웃 | 인라인 저장 |
| 프로필 열기 | 이름 클릭 | 프로필 모달 오픈 |
| 전화번호 복사 | 전화번호 클릭 | 클립보드 복사 + 토스트 |
| 출석 상세 | 출석/불참/대타 셀 클릭 | 캘린더 모달 오픈 |

## 7. 현재 제약사항 (Known Limitations)

1. **데이터**: 60명 리더 랜덤 생성 (Mock), 백엔드 미연동
2. **팀 탭 필터**: UI만 구현, 실제 필터링 로직 미완성 (data-team 속성만 존재)
3. **드래그 정렬**: 핸들 UI만 존재, 드래그앤드롭 미구현
4. **캘린더 모달**: `openCalendarModal()` 호출만 존재, 모달 구현 미완성
5. **SMS 모달**: `openSmsModal()` 호출만 존재, 모달 구현 미완성
6. **액션 드롭다운**: 메뉴 항목 미구현 (점3개 버튼만 존재)
7. **팀원 추가**: 버튼만 존재, 추가 기능 미구현
8. **검색**: UI만 존재, 실시간 필터 JS 로직 미구현
