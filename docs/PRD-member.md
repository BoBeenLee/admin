# PRD: 멤버 팀 출석부 — member.html

> 하남 지점 어드민 > 멤버 팀 관리 페이지 제품 요구사항 정의서

---

## 1. 개요

| 항목 | 내용 |
|------|------|
| **페이지명** | 멤버 팀 출석부 |
| **파일** | `member.html` |
| **역할** | 어학원 학생(멤버)의 팀 배정, 출석, 멤버십, 리딩 레벨, 알림을 종합 관리하는 출석부 |
| **주요 사용자** | 지점장, 매니저 |
| **진입 경로** | 사이드 네비게이션 "멤버 팀" |

## 6. 데이터 생성 전략

member.html은 **Seeded PRNG (Mulberry32)** 를 사용하여 결정적 Mock 데이터를 생성합니다.

```
시드 = hash(teamKey + "_" + level)
→ 동일 팀+레벨 조합은 항상 동일한 멤버 30명 생성
→ 팀당 5레벨 × 30명 = 150명
→ 전체 10팀 × 150명 = 1,500명 (중복 이름 방지)
```

이를 통해 페이지 새로고침 시에도 동일한 데이터가 유지됩니다.

## 8. leader.html과의 차이점

| 항목 | leader.html | member.html |
|------|------------|-------------|
| **대상** | 리더 (튜터) 60명 | 멤버 (학생) 1,500명 |
| **팀 구조** | 요일+시간 단위 (영 월 11시) | 일정+시간 단위 (영어 월수 11시) |

## 9. 현재 제약사항 (Known Limitations)

1. **데이터**: Seeded PRNG Mock (팀당 레벨당 30명), 백엔드 미연동
2. **팀 간 멤버 공유**: 멤버가 2개 팀에 배정될 수 있으나, 실제 교차 참조 미구현
3. **검색**: UI 존재, 실시간 필터 JS 부분 구현 (팀 전환 시 동작)
4. **SMS 모달**: `openSmsModal()` 호출만 존재, 모달 미구현
5. **액션 드롭다운**: 점3개 버튼만 존재, 메뉴 항목 미구현
6. **팀원 추가**: 버튼만 존재, 기능 미구현
7. **드래그 정렬**: 핸들만 존재, 드래그앤드롭 미구현
8. **알림 자동 감지**: 알림은 데이터 생성 시 랜덤 할당, 실시간 조건 기반 감지 아님
9. **히스토리**: 프로필 모달의 히스토리는 템플릿 기반 (3개 고정 항목)
